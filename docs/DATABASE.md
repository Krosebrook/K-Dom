# Database Reference — K-Dom

## Overview

K-Dom has no database server. All persistent state is stored in the browser's **localStorage**. There is exactly one storage key.

| Storage | Key | Type |
|---------|-----|------|
| `localStorage` | `kingdom_builder_save_v1` | JSON-serialized `SaveData` object |

---

## Save Data Schema

### Root Object: `SaveData`

```typescript
interface SaveData {
  grid: Grid;           // 34×34 tile array
  stats: KingdomStats;  // All kingdom statistics
  persona: AdvisorPersona;  // Selected advisor persona string
  version: number;      // Currently always 1
}
```

### `Grid` — `TileData[][]`

A 34×34 matrix (34 rows × 34 columns). Indexed as `grid[y][x]`.

```typescript
type Grid = TileData[][];

interface TileData {
  x: number;         // Column index (0–33)
  y: number;         // Row index (0–33)
  buildingType: BuildingType;  // Enum string value (e.g., "Hovel")
  variant?: number;  // Unused field — always undefined
  rotation?: number; // 0–3 (×90° increment)
  isOpen?: boolean;  // Gatehouse/Drawbridge open state
  isLocked?: boolean; // Gatehouse/Drawbridge lock state
  isWet?: boolean;   // Moat/Drawbridge hydration state
  level: number;     // Building upgrade level (1 = default, 2, 3+)
}
```

#### `BuildingType` Values Stored
```
"None" | "Path" | "Hovel" | "Market" | "Farm" | "Keep" | "Barracks" | "Wall" | "Gatehouse" | "Tower" | "Moat" | "Drawbridge"
```

---

### `KingdomStats`

```typescript
interface KingdomStats {
  gold: number;       // Treasury balance (starts at 800)
  subjects: number;   // Total population
  soldiers: number;   // Military foot soldiers
  officers: number;   // Military officers (lead squads)
  day: number;        // Game day counter (starts at 1)
  defense: number;    // Calculated defense rating
  happiness: number;  // 0–100 happiness score
  season: Season;     // "Spring" | "Summer" | "Autumn" | "Winter"
}
```

---

### `AdvisorPersona` Values Stored
```
"The Steward" | "The Warlord" | "The Merchant" | "The Architect"
```

## Storage Size Estimate

| Component | Entries | Approx. bytes per entry | Total |
|-----------|---------|------------------------|-------|
| Grid tiles | 1,156 (34×34) | ~60 bytes | ~69 KB |
| Stats | 1 | ~100 bytes | ~100 bytes |
| Persona | 1 | ~15 bytes | ~15 bytes |
| **Total** | | | **~70 KB** |

Well within localStorage limits (typically 5–10 MB per origin).

## Schema Version

The `version` field in `SaveData` is currently always `1`. No migration logic exists. If the schema changes in a future version, saves from v1 may fail to load if required new fields are missing. The current `loadGame()` in `App.tsx` does not validate the version field.

## Operations

### Read (Load)
```typescript
// App.tsx:135-148
const loadGame = () => {
    try {
        const raw = localStorage.getItem('kingdom_builder_save_v1');
        if (raw) {
            const data = JSON.parse(raw) as SaveData;
            setGrid(data.grid);
            setStats(data.stats);
            setPersona(data.persona || AdvisorPersona.Balanced);
            setGameStarted(true);
        }
    } catch (e) {
        console.error("Failed to load save", e);
    }
};
```

### Write (Auto-save)
```typescript
// App.tsx:107-116
const saveGame = useCallback(() => {
    if (!gameStarted) return;
    const data: SaveData = {
        grid: gridRef.current,
        stats: statsRef.current,
        persona: personaRef.current,
        version: 1
    };
    localStorage.setItem('kingdom_builder_save_v1', JSON.stringify(data));
}, [gameStarted]);
// Called every 5 seconds via setInterval
```

### Delete
Not exposed in the UI. A user can clear their save manually via browser DevTools:
```javascript
localStorage.removeItem('kingdom_builder_save_v1');
```

## Data Not Persisted

The following game state is **ephemeral** (lost on page refresh):

| Data | Reason |
|------|--------|
| Chat history (`ChatMessage[]`) | Conversational context; intentionally not saved |
| News feed (`NewsItem[]`) | Regenerated by AI each session |
| Active event (`GameEvent`) | Modal state; dismissed before save typically |
| Floating texts (`FloatingText[]`) | Transient UI feedback |
| Unit positions and AI state | Unit system reinitializes from stats on each load |
| AI goal (`AIGoal`) | New goal generated on each game start |

---

## Integrity Notes

- No checksums or signatures on saved data — a user can manually edit their save in DevTools
- `persona` field has a fallback: if missing from the save, `AdvisorPersona.Balanced` is used
- No version migration: loading a save from a future schema version with new required fields will likely cause runtime errors
